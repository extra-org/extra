title: Quickstart
description: Get Agent Engine running in two modes — engine only, or with conversation persistence and an embeddable widget.

## Prerequisites

- Docker (recommended) **or** Python 3.11+
- An API key for your LLM provider (Anthropic, Bedrock, etc.)

---

## Path 1 — Engine Only

The stateless engine exposes `/invoke` and `/stream`. No database, no widget. Good for embedding into your own backend.

### 1. Write your spec

Create `agents.yml` (see [YAML Reference](/docs/yaml-spec) for all fields):

```yaml
system:
  name: "Support Bot"

defaults:
  model:
    provider: anthropic
    name: claude-sonnet-4-6

orchestrators:
  router:
    description: "Routes the user to the right department."
    prompts:
      orchestrator: "prompts/router/orchestrator.md"

agents:
  orders_agent:
    description: "Handles order status and tracking questions."
    prompts:
      system: "prompts/orders/system.md"

  returns_agent:
    description: "Handles return requests and refunds."
    prompts:
      system: "prompts/returns/system.md"

graph:
  router:
    orders_agent:
    returns_agent:
```

### 2. Generate plugin stubs

```bash
docker run --rm \
  -v $(pwd):/workspace \
  -w /workspace \
  ghcr.io/amitavital/extra:latest \
  generate --config agents.yml
```

This creates the `plugins/` folder with stub files for your resolvers, tools, and hooks. Fill them in with your business logic.

### 3. Validate

```bash
docker run --rm \
  -v $(pwd):/workspace \
  -w /workspace \
  ghcr.io/amitavital/extra:latest \
  validate agents.yml
```

Fully offline — no LLM calls, no network. Exits non-zero on any error.

### 4. Run the engine

```bash
docker run -d \
  -p 8080:8080 \
  -v $(pwd):/workspace \
  -w /workspace \
  -e ANTHROPIC_API_KEY=sk-... \
  ghcr.io/amitavital/extra:latest \
  serve --config agents.yml
```

Call it:

```bash
curl -X POST http://localhost:8080/invoke \
  -H "Content-Type: application/json" \
  -d '{"messages": [{"role": "user", "content": "Where is my order?"}]}'
```

## Path 2 — Conversation Layer + Widget

`agent-manager` adds conversation persistence, multi-turn sessions, SSE streaming, and the embeddable chat widget on top of the engine.

### 1. Same `agents.yml` as above

No changes to your spec. `agent-manager` wraps the engine.

### 2. Run agent-manager

```bash
docker run -d \
  -p 8100:8100 \
  -v $(pwd):/workspace \
  -w /workspace \
  -e ANTHROPIC_API_KEY=sk-... \
  ghcr.io/amitavital/extra:latest \
  agent-manager --config agents.yml --port 8100
```

This starts the conversation API (`/conversations`, `/conversations/{id}/messages`) and serves the widget at `/widget.js`.

### 3. Embed the widget

Drop this into any HTML page — React, Vue, plain HTML, doesn't matter:

```html
<script type="module" src="http://localhost:8100/widget.js"></script>
<agent-chat
  title="Support"
  color="#6366f1"
  greeting="Hi! How can I help you today?"
  mode="floating"
  position="bottom-right">
</agent-chat>
```

That's it. The widget handles sessions, streaming, and conversation history automatically.

<Tip>
  See [Widget](/docs/widget) for all `<agent-chat>` attributes, inline vs floating mode, and how to embed it in frameworks like React or Next.js.
</Tip>

### What's different between the two modes

| | Engine Only | With agent-manager |
|---|---|---|
| Conversation history | You own it | Built-in (SQLite/Postgres) |
| Sessions | You manage | Auto-managed |
| Streaming | SSE via `/stream` | SSE built-in |
| Widget | No | Yes |
| Port | 8080 | 8100 |

## Local development (no Docker)

```bash
git clone https://github.com/amitavital/extra.git
cd extra
python3 -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"

# Engine
agentctl validate agents.yml
agentctl serve --config agents.yml

# Conversation layer
agent-manager --config agents.yml --port 8100
```
