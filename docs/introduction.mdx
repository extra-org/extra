## What is Extra Agent Engine?

Extra Agent Engine is an open-source platform for building and running AI agent systems declaratively.

You describe your agents, orchestrators, tools, MCP servers, and their connections in a YAML file.
The engine validates, compiles, and runs that system — without you writing plumbing code.

```yaml
system:
  name: "My AI System"

orchestrators:
  router:
    description: "Routes user requests."
    prompts:
      orchestrator: "prompts/router.md"

agents:
  support_agent:
    description: "Handles support questions."
    prompts:
      system: "prompts/support.md"
    mcps: [helpdesk]

graph:
  router:
    support_agent:
```

## Why Extra Agent Engine?

Building multi-agent systems by hand means re-implementing the same plumbing every time:
prompt rendering, agent routing, tool wiring, MCP auth, and tracing.

Extra Agent Engine moves that plumbing into a reusable runtime so you focus on **what** your system does,
not **how** it runs.

## Key Features

- **YAML-first** — orchestrators, agents, prompts, resolvers, tools, and graph topology in one file
- **Runtime hooks** — trusted lifecycle code for auth, policy, and audit — never seen by the LLM
- **MCP auth** — `before_mcp_request` hooks inject headers per request; tokens never reach the model
- **Conversation persistence** — `agent_manager` layer with SQLite/Postgres and SSE streaming
- **Embeddable widget** — Shadow DOM-isolated `<agent-chat>` custom element
- **Observability** — pluggable LangChain callbacks; Langfuse tracing auto-enables via env vars
- **Docker-ready** — official image, defaults to `serve` mode
