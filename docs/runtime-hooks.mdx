## What are hooks?

Hooks are Python code that runs automatically at specific moments in the request lifecycle. They are **never exposed to the model** — it can't see them, name them, or call them. That's what separates a hook from a tool.

They're the right place for:

- **Auth** — inject credentials before MCP requests
- **Audit** — log tool calls and run outcomes
- **Policy** — block or reshape a tool result before it reaches the model
- **Context enrichment** — attach identity or metadata to a run

---

## Hooks vs. tools

| | Tool | Hook |
|---|---|---|
| Called by | The model, when it decides to | The engine, automatically |
| Visible to the model | Yes | **Never** |
| Purpose | Do work the model asked for | Auth, audit, policy |

---

## Lifecycle points

Every row below is a moment the engine will call your hook, if you've declared one for it.

### Engine lifecycle

Fire once per engine process, not per request.

| Hook point | Fires | Can modify | Typical use |
|---|---|---|---|
| `on_engine_start` | Once, before any request is served | No — observe only | Validate required secrets/connections exist at boot |
| `on_engine_stop` | Once, during shutdown (best-effort) | No — observe only | Cleanup, flush buffers |

### Run lifecycle

Fire once per request.

| Hook point | Fires | Can modify | Typical use |
|---|---|---|---|
| `on_run_start` | Before the graph executes | **Yes** — may return an updated run context | Attach identity/metadata to the run |
| `on_run_end` | After a successful response is ready | No — observe only | Log run outcome |
| `on_run_error` | When a request fails, for any reason | No — never masks the original error | Alert, log failures |

### Tool lifecycle

Fire around every local or MCP tool call.

| Hook point | Fires | Can modify | Typical use |
|---|---|---|---|
| `before_tool_call` | Right before any tool runs | No — observe only | Audit, policy checks |
| `after_tool_call` | After a tool call succeeds | No — observe only | Audit logging |
| `transform_tool_result` | After a tool succeeds, before its result reaches the model | **Yes** — may return an updated result | Truncate, redact, or normalize a result |
| `on_tool_error` | When a tool call raises an error | No — observe only | Alert, log failures |

### MCP lifecycle

Fire around every outgoing MCP HTTP request.

| Hook point | Fires | Can modify | Typical use |
|---|---|---|---|
| `before_mcp_request` | Right before an MCP HTTP request goes out — every operation: connect, list tools, call tool | **Yes** — may return the request with updated headers | Inject auth headers ([most common use](#mcp-auth)) |
| `after_mcp_response` | After every MCP HTTP response, success or error status | No — observe only | Log latency, status codes |

## Declaring a hook

```yaml
hooks:
  before_mcp_request:
    - plugin: mcp_auth
      method: before_mcp_request
      config:
        credential_env: INTERNAL_MCP_SECRET

  after_tool_call:
    - plugin: audit
      method: record_tool_call
      failure_policy: warn
```

Each lifecycle point above can take a list of hook entries — the fields for one entry:

<ParamField path="plugin" type="string">
  Logical plugin id, resolved through `plugins/plugins.toml`. Used together with `method`. Mutually exclusive with `ref`.
</ParamField>

<ParamField path="method" type="string">
  The method name to call on the resolved plugin class. Required when using `plugin`.
</ParamField>

<ParamField path="ref" type="string">
  A direct import path to the hook function — an advanced alternative to `plugin` + `method`, for manual/legacy wiring. Mutually exclusive with `plugin` + `method`.
</ParamField>

<ParamField path="config" type="object" default="none">
  Arbitrary settings passed to the hook on every call.

  <Warning>
    Never put secrets directly here. Store the secret in an environment variable and reference the variable's *name* — the hook code reads the real value from the environment at call time.
  </Warning>
</ParamField>

<ParamField path="failure_policy" type="string" default="fail">
  What happens if the hook itself raises an error.

  <Expandable title="possible values">
    <ResponseField name="fail" type="string">
      **Default.** The operation is aborted and the request fails. Use this for anything security-critical, like auth — a broken auth hook should never fail open.
    </ResponseField>
    <ResponseField name="warn" type="string">
      The error is logged and the request continues as if the hook had succeeded. Use this for best-effort work like audit logging, where a logging failure shouldn't block the user.
    </ResponseField>
  </Expandable>
</ParamField>

## MCP auth {#mcp-auth}

The most common hook. Adds auth headers to every outgoing MCP request so private servers work — without putting tokens in prompts or YAML.

**YAML:**
```yaml
hooks:
  before_mcp_request:
    - plugin: mcp_auth
      method: before_mcp_request
      config:
        credential_env: INTERNAL_MCP_SECRET
```

**`plugins/plugins.toml`:**
```toml
[hooks.plugins]
mcp_auth = "plugins.hooks.mcp_auth:McpAuthHook"
```

**`plugins/hooks/mcp_auth.py`:**
```python
import os
from agent_engine.runtime.hooks import HookInvocation, McpRequestContext

class McpAuthHook:
    def __init__(self, config=None):
        self.config = config

    async def before_mcp_request(self, event: HookInvocation) -> McpRequestContext:
        request = event.payload_as(McpRequestContext)
        credential = os.environ[dict(event.config or {})["credential_env"]]
        return request.with_headers({"Authorization": f"Bearer {credential}"})
```

The token is read from the environment at request time. It never appears in YAML, prompts, or logs.

### Token exchange

If you need to exchange the user's inbound token for an MCP-scoped one:

```python
async def before_mcp_request(self, event: HookInvocation) -> McpRequestContext:
    request = event.payload_as(McpRequestContext)
    inbound_token = event.run_context.auth_context.inbound_access_token
    mcp_token = await exchange_token(inbound_token, audience="mcp-service")
    return request.with_headers({"Authorization": f"Bearer {mcp_token}"})
```

### HMAC signing

```python
import hashlib, hmac, os, time

async def before_mcp_request(self, event: HookInvocation) -> McpRequestContext:
    request = event.payload_as(McpRequestContext)
    secret = os.environ["HMAC_SECRET"].encode()
    ts = str(int(time.time()))
    sig = hmac.new(secret, f"{ts}:{request.url}".encode(), hashlib.sha256).hexdigest()
    return request.with_headers({"X-Timestamp": ts, "X-Signature": sig})
```

## Audit hook example

```python
class AuditHook:
    async def record_tool_call(self, event: HookInvocation) -> None:
        call = event.payload_as(ToolCallContext)
        await audit_log.write({
            "run_id": event.run_context.run_id,
            "tool": call.tool_name,
            "provider": call.provider,
            "status": call.status,
            "latency_ms": call.latency_ms,
        })
```

Set `failure_policy: warn` on audit hooks — a logging failure shouldn't abort the user's request.

## Truncating tool results

Use `transform_tool_result` to shorten large MCP responses before they reach the model:

```python
async def trim_result(self, event: HookInvocation) -> ToolResultContext:
    result = event.payload_as(ToolResultContext)
    if isinstance(result.result, str) and len(result.result) > 2000:
        return result.with_result(result.result[:2000] + "\n… [truncated]")
    return result
```

## Passing identity into hooks

The embedding application can pass per-request identity through the run context:

```python
from agent_engine.runtime.hooks import AuthContext, RunContext

ctx = RunContext(
    user_id="user-123",
    organization_id="org-7",
    auth_context=AuthContext(
        inbound_access_token=request.headers["Authorization"],
        scopes=("support:read",)
    )
)
result = await engine.run(message, context=ctx)
```

Inside a hook, access it through `event.run_context`:

```python
user_id = event.run_context.user_id
token = event.run_context.auth_context.inbound_access_token
```

## Security

Hooks are trusted in-process code. The engine never logs auth headers, tokens, HMAC signatures, or hook config values. Your hook code must follow the same rule — never log secrets.
