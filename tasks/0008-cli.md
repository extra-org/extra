# Task 0008 — CLI

## Goal

Provide a command-line interface to validate, compile, and inspect an
YAML config, and to run a single request locally against a built runtime.

## Context

The CLI is a thin surface over existing layers (validation, compiler, runtime).
It must construct the `RuntimeEngine` **once** per invocation and create an
`ExecutionContext` per request — never a runtime per request.

**Read first:** `AGENTS.md`, `.ai/skills/project-architecture.md`,
`.ai/skills/runtime-engine.md`, `docs/RUNTIME_LIFECYCLE.md`.

## Scope

- Implement CLI commands: `validate`, `compile` (inspect graph), and a local
  `run`/`invoke` for a single request.
- Wire to existing layers; do not reimplement them.

## Files allowed to change

- `src/agentplatform/cli/**`
- `pyproject.toml` (only to register a console entry point)
- `tests/cli/**`

## Requirements

- `validate <file>`: loads and validates; prints structured errors or success.
- `compile <file>`: builds the `CompiledAgentGraph` and prints a readable summary
  (orchestrators, agents, graph, bindings).
- `run`/`invoke`: builds the runtime once, creates one `ExecutionContext`, and
  returns the response + a trace summary.
- Clear exit codes (non-zero on validation/compile failure).
- No business logic, no secrets; secrets referenced via env, redacted in output.

## Out of scope

- The HTTP API (task 0009).
- Deployment (0010).
- New runtime/validation/compiler behavior (reuse existing layers).

## Acceptance criteria

- [ ] `validate`, `compile`, and `run`/`invoke` commands work end-to-end on a
      sample spec.
- [ ] Runtime is built once per invocation; context is per request.
- [ ] Failure paths return non-zero exit codes with clear messages.
- [ ] No secrets printed; tokens redacted.
- [ ] Tests cover each command (success + failure).
- [ ] `make check` passes.

## Commands to run before finishing

```bash
make check
```

## Expected final report

Use the AGENTS.md §9 format. Confirm the CLI reuses existing layers and respects
the runtime lifecycle. Recommend task 0009 next.
